lcmap.data.ingest

Funtions related to the creation of tiles from ESPA XML metadata and associated rasters (e.g. GDAL datasets)

+fill

(+fill band)

Make a fill buffer used to detect no-data tiles

+locate

(+locate band)

Make a raster to projection point transformer function.

+spec

(+spec db band)

Retrieve a spec (for the given UBID) and add it to the band. This assumes only one tile-spec will be found. If multiple tile-specs exists, behavior is undefined.

checksum

(checksum tile)

Produce a checksum for tile data. Useful for building a secondary inventory of ingested data and verifying multiple runs produce the same tile data. This relies on configuring underlying logging (currently); see resources/logback.xml for details.

conforms?

(conforms? band)

PLACHOLDER. True if the referenced raster matches the band’s tile-spec. This ensures the raster is the same projection and that the boundaries precisely align to the tile-specs grid values.

dataset->tiles

(dataset->tiles tile-xf dataset x-step y-step)

Create sequence of tile from dataset referenced by band.

fill-buffer

Create a buffer memoized on data-size and data-fill.

fill?

(fill? tile)

True if the tile is comprised entirely of fill values

get-hash

(get-hash tile)

Calculate MD5 checksum of tile

int16-fill

(int16-fill data-size data-fill)

Produce a buffer used to detect INT16 type buffers containing all fill data.

locate

(locate tile)

Use band’s locator to turn a raster point to projection point.

locate-fn

(locate-fn band)

Build projection coordinate point calculator for GDAL dataset.

process-band

(process-band db band)

Saves band as tiles.

process-scene

(process-scene db scene-dir)

Saves all bands in dir referenced by path.

process-tile

(process-tile db tile)

Isolates all side-effect related behavior performed on each tile

save

(save db tile)

Save a tile. This function should be used for all saving that needs to happen (in batch) when processing a tile. Currently, this only inserts tile data, but it will soon update a scene inventory too.

scene->bands

(scene->bands path band-xf)

Create sequence of from ESPA XML metadata.
